Consider a medical AI assistant that learns at 10:00~AM on Tuesday that a patient's symptoms began two days prior.
To correctly answer ``What was the patient's status on Sunday morning?'', the system must distinguish \emph{when the symptoms started} (the \textbf{event time}) from \emph{when it recorded this fact} (the \textbf{record time}).
This bitemporal distinction, well-established in temporal database theory~\citep{snodgrass1999tsql2,jensen1999temporal}, is pervasive: financial trades execute Monday but settle Wednesday, news events occur Thursday evening but are reported Friday morning, scientific findings are established in 2022 but published in 2024.
A memory system that conflates these two timestamps will make incorrect temporal inferences.

Despite this practical importance, no existing benchmark systematically evaluates whether LLM agent memory systems can reason about both dimensions simultaneously.
Temporal reasoning benchmarks (e.g., \textsc{TimE}~\citep{time2025}, \textsc{ETRQA}~\citep{etrqa2024}, \textsc{Test of Time}~\citep{testoftime2025}) focus on LLM reasoning over static text, not over dynamically evolving memory stores.
Memory system benchmarks (e.g., LoCoMo~\citep{locomo2024}, LongMemEval~\citep{longmemeval2024}, MemoryAgentBench~\citep{memoryagentbench2025}) include temporal questions but treat time as a single flat dimension, without separating event occurrence from information acquisition.
This gap means we cannot diagnose whether current memory backends fail on temporal questions because of \emph{retrieval limitations} or \emph{reasoning limitations}, nor identify which types of temporal reasoning are hardest.

We present \textbf{BiTempQA}, a diagnostic benchmark that fills this gap.
BiTempQA is designed around a core insight from bitemporal data modeling: every fact has two independent temporal dimensions.
Our benchmark contains \textbf{1,536 Chinese QA pairs (308 held-out test)} spanning \textbf{10 scenario types} across 6 domains (corporate, academic, social, fictional, historical, scientific), with \textbf{9 question types} at \textbf{3 difficulty levels}.
Each memory write carries explicit \texttt{event\_time} and \texttt{record\_time} annotations, and 56.5\% of questions require reasoning about \emph{both} timestamps simultaneously.
We evaluate five memory backends (FAISS, BM25, Naive RAG, Simple Knowledge Graph, and ChromaDB) through a unified Retrieve$\to$Generate$\to$Judge pipeline.

Our contributions are as follows.
First, we introduce the \textbf{first dual-timestamp diagnostic benchmark} for memory systems, with explicit event/record time annotations on every memory write and structured metadata on every question.
Second, we provide \textbf{evidence that record-time reasoning is genuinely harder} ($p{=}0.023$ in mixed-effects regression controlling for question type, difficulty, and scenario), confirming the benchmark's diagnostic validity.
Third, we deliver \textbf{actionable diagnostic findings}: retrieval (not reasoning) is the primary failure bottleneck ($\sim$55\%); system confidence is anti-calibrated (AUROC${}<0.5$); memory backends are highly complementary (oracle +9.8pp over best single system).
Fourth, we present \textbf{cross-benchmark analysis} with LoCoMo~\citep{locomo2024} revealing that graph-based memory excels at adversarial robustness (97.5\%) but struggles with temporal reasoning (37.5\%).
