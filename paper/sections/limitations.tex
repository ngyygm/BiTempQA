\section{Limitations}
\label{sec:limitations}

We note several limitations of the current benchmark and evaluation setup:

\paragraph{Scale and language.}
% 中文注释：这里只统一数学符号和字母间距，不改变 limitation 的含义。
BiTempQA contains 1,536 items (308 test items) and is currently limited to Chinese. While this scale supports diagnostic analysis (bootstrap CIs $\leq 8$pp for major categories), smaller subcategories (record-only $n = 24$, retraction $n = 28$) have wide confidence intervals that limit strong quantitative claims. Expanding the test set to 600--1000 items and adding English translations are important directions for future work.

\paragraph{LLM-generated data.}
All QA pairs are generated by Qwen2.5-72B-Instruct and have not yet undergone human expert validation.
We mitigate this through (1) structured generation with strict output schemas, (2) automated consistency checks, and (3) LLM Judge cross-validation ($\kappa = 0.455$).
However, some items may still contain ambiguous or imperfectly specified answers.
Human validation on a stratified sample (100--150 items, 2 native annotators) is therefore a high-priority next step.

\paragraph{ChromaDB confound.}
ChromaDB uses a different default embedding model (all-MiniLM-L6-v2, 22M parameters) than the other vector systems (SiliconFlow API, multilingual).
Its poor performance (41.2\%) likely reflects this configuration difference rather than a fundamental limitation of persistent vector databases.
We therefore include ChromaDB for completeness but do not rely on it for the paper's primary comparative claims.

\paragraph{Train/test scenario overlap.}
All 112 test scenarios also appear in the training set, albeit with different questions. While this mirrors common reading-comprehension setups (same passages, different questions), it also means that systems fine-tuned on the training data may benefit from scenario familiarity. Future versions should use fully held-out scenarios for the test set.

\paragraph{Backend scope.}
% 中文注释：这里继续保持 Times 系正文风格，不使用额外特殊字体。
We evaluate memory storage and retrieval backends rather than full agent memory systems (e.g., Mem0, Zep, and Graphiti). These production systems incorporate additional components, such as memory consolidation, summarization, and conflict resolution, that may affect bitemporal reasoning. Evaluating such systems remains ongoing work.

\paragraph{LLM Judge leniency.}
DeepSeek-V3 is more lenient than Qwen2.5-72B (80\% vs.\ 60\% accuracy on a 50-item sample), with 11 of 12 disagreements favoring acceptance. Reported accuracies may therefore be somewhat higher than those obtained with a stricter judge, although relative system rankings are likely to be more stable.
