\subsection{Data Construction Pipeline}

The BiTempQA v3 dataset is constructed in four stages:

\begin{enumerate}
\item \textbf{ChronoQA extraction:} 200 bitemporal candidates selected from ChronoQA with gap $\geq 1$ day, balanced across gap ranges and question types.
\item \textbf{Curated scenarios:} 15 manually constructed scenarios from verified public events across 6 domains.
\item \textbf{LLM augmentation:} 118 template-generated scenarios covering rare types not found in real data (contradictory information, future-dated information, entity identity resolution).
\item \textbf{Merge and split:} All 333 scenarios combined, QA pairs extracted, shuffled (seed=42), and split 70/10/20 into train/dev/test. Test set contains only real-source scenarios.
\end{enumerate}
