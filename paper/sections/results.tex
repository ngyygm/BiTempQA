\section{Results}

\subsection{Overall Performance}
Table~\ref{tab:overall} presents overall accuracy by system.
FAISS achieves the highest accuracy at 75.6\%, followed closely by Naive RAG (74.0\%) and BM25 (73.4\%).
Bootstrap 95\% CIs overlap substantially among the top three, and pairwise McNemar's tests show no significant differences ($p > 0.3$).
Simple KG (67.2\%) and ChromaDB (41.2\%) perform significantly worse than FAISS ($p < 0.01$ and $p < 0.001$, respectively).
ChromaDB uses a different default embedding model (all-MiniLM-L6-v2) than the other vector-based systems; we include it for completeness but note this confound.

Table~\ref{tab:temporal_breakdown} shows accuracy by temporal requirement.
Record-only questions average 60.8\% compared with 69.1\% for event-only questions, yielding an 8.3pp gap that appears consistently across all five systems.
To examine whether this pattern can be explained by other dataset factors, we fit a mixed-effects logistic regression with scenario random intercepts, controlling for system identity, question type, difficulty level, and temporal requirement flags.

The record-time coefficient ($\beta = -0.079$, $p = 0.023$) remains significant after controlling for the other factors in the model (Table~\ref{tab:regression}), indicating that record-time requirements are associated with greater difficulty.
Notably, difficulty levels (L2/L3) are not independently predictive after controls ($p = 0.103$, $p = 0.278$), suggesting that the observed difficulty gradient is largely mediated by the distribution of harder question types at higher levels.

Figure~\ref{fig:heatmap} reveals substantial system $\times$ type interactions.
FAISS leads on complex temporal (87.1\%) and counterfactual (53.8\%); BM25 excels at point-in-time (85.7\%) and multi-hop (83.3\%); Simple KG tops period query (88.9\%).
The 40.6pp spread across question types (counterfactual: 48.7\% to temporal order: 89.3\%) suggests that the typology captures meaningfully different challenges.

\subsection{Difficulty Calibration}
% 中文注释：这里只统一箭头和符号间距，不改变结论或图表大小。
All five systems show monotonic L1 $\rightarrow$ L3 degradation, but the magnitude of the drop varies substantially: FAISS declines by only 2.9pp, whereas BM25 declines by 16.6pp (Figure~\ref{fig:difficulty}).
FAISS's dense representations appear to maintain semantic relevance even under complex compositional demands, while BM25's keyword matching deteriorates sharply when query--document lexical overlap decreases.

\subsection{System Complementarity}
An oracle ensemble (counting an item as correct if any system answers it correctly) reaches 85.4\%, a 9.8pp gain over the best single system, suggesting that the memory backends have substantially non-overlapping error profiles (Figure~\ref{fig:complementarity}).
FAISS-ChromaDB Jaccard overlap is only 0.446 (lowest), while FAISS-SimpleKG is 0.746.
Simple KG uniquely solves 6 items that FAISS misses, and ChromaDB uniquely solves 6, despite its low overall accuracy.
Forty-five items (14.6\%) are answered incorrectly by all systems, indicating a substantial residual error set under the current evaluation setup.

\subsection{Cross-Benchmark Validation with LoCoMo}
Table~\ref{tab:locomo} compares BiTempQA results with LoCoMo evaluation.
% 中文注释：这里只整理字体和符号间距，保留原有对比结论。
The clearest cross-benchmark asymmetry appears in Simple KG: it reaches 97.5\% adversarial accuracy on LoCoMo (best among the evaluated systems, +34pp above average), but only 37.5\% on LoCoMo temporal reasoning and 67.2\% on BiTempQA.
This pattern is consistent with a trade-off in which graph structures help avoid adversarial hallucination by returning little or no context for non-existent facts, but offer less retrieval breadth for temporal reasoning.
