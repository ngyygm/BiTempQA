# S07: 知识撤回/纠正 (Knowledge Retraction)

## 场景描述
已记录的事实被后续信息明确纠正或撤回。Agent需要能够标记旧信息为"已失效"并使用新信息。

## 核心双时间轴测试点
- 版本回溯能力
- 知识失效的传播（依赖被撤回知识的其他事实是否受影响）
- 撤回时间与原事件时间的关系

## 示例模板
```
领域: [corporate/scientific/news]
撤回类型: [错误纠正/政策变更/事实更新/声明撤回]

w1: 在[T1]记录了事实F               → event_time=T1, record_time=T1
w2: 在[T2]确认了基于F的推理C         → event_time=T2, record_time=T2
w3: 在[T3]撤回F，声明F是错误的，正确的是F' → event_time=T3, record_time=T3
```

## 关键约束
- 必须有明确的撤回/纠正事件
- 撤回前的信息需要有多个write依赖（形成依赖链）
- 至少1个QA对需要验证"撤回后是否仍使用旧信息"
- 撤回事件需要可被语义识别（如"更正""撤回""实际是"等关键词）

## QA生成指引
- Level 1: "在时间T（撤回后），Agent认为事实F是什么？"（验证撤回生效）
- Level 2: "基于被撤回的事实F，在T2时Agent得出的结论C在T3后是否仍然有效？"（依赖传播）
- Level 3: "如果Agent不支持知识撤回，在T3之后它会给出什么错误答案？"
