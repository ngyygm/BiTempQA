# S06: 实体身份解析 (Entity Identity Resolution)

## 场景描述
同一实体在不同时间以不同名称/标识出现，需要跨时间进行实体对齐。包括名称变更、组织合并、品牌更名等。

## 核心双时间轴测试点
- 跨时间实体对齐（不同名称 = 同一实体）
- 身份变更的时间点精确性
- 对齐错误导致的事实归属错误

## 示例模板
```
领域: [corporate/social/fictional]
身份变更类型: [改名/合并/分拆/别名]

w1: [实体A（旧名）]在[T1]具有属性P     → event_time=T1
w2: [实体A]在[T2]更名为[实体A'（新名）]  → event_time=T2
w3: [实体A'（新名）]在[T3]具有属性P'    → event_time=T3
```

## 关键约束
- 同一实体至少出现2个不同名称
- 需要有明确的身份变更事件
- 查询需要使用不同名称指代同一实体
- 至少1个QA对需要跨名称查询（用旧名称查询新属性，或反之）

## QA生成指引
- Level 1: "[新名]在时间T3的属性是什么？"（直接查询）
- Level 2: "在[T1,T3]期间，[旧名/新名]的属性变化是什么？"（跨名称版本追踪）
- Level 3: "如果Agent没有识别出[旧名]和[新名]是同一实体，对T2之后的查询会产生什么错误答案？"
